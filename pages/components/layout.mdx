# Layout

<br />

<FadeIn>
인봄 플랫폼의 모든 페이지를 감싸는 레이아웃 컴포넌트 모음이에요.

Layout 하나로 페이지 전체 구조를 통일해요. <br className="br" />
Header와 NavBar는 Layout 내부에 고정되고, <br className="br" />
그 사이 `<main>` 영역에 각 페이지의 children이 렌더링돼요.

```tsx
const Layout = ({ children }) => {
  return (
    <div className={styles.layout}>
      <Header />
      <main className={styles.content}>{children}</main>
      <NavBar />
    </div>
  );
};
```

Footer는 Layout에 포함되지 않아요. <br className="br" />
스크롤을 모두 내린 후 페이지 하단에 자연스럽게 위치해야 하기 때문에, <br className="br" />
필요한 페이지에서 children 안에 직접 배치해요.

```tsx
// 페이지에서 직접 사용
<Layout>
  {/* 페이지 콘텐츠 */}
  <Footer />
</Layout>
```

Header, NavBar, MobileMenu — 세 컴포넌트가 Layout과 함께 동작하며 <br className="br" />
인증 상태, 사용자 권한(ROLE), 현재 경로를 공유해 유기적으로 연결돼요.

</FadeIn>

## Header

<br />

<FadeIn>
데스크탑에서 상단에 고정되는 헤더예요.

로그인 상태에 따라 우측 영역이 달라져요. <br className="br" />
비로그인이면 로그인 / 회원가입 / 견적요청 버튼이 보이고, <br className="br" />
로그인 상태면 알림 아이콘과 프로필 버튼이 나타나요. <br className="br" />

사용자 권한(`ROLE`)에 따라 프로필 버튼이 이동하는 경로도 달라요. <br className="br" />
일반회원(`U`)은 마이페이지, 관리자(`A`)는 어드민, 사업자(`B`)는 사업자 마이페이지로 연결돼요.

현재 경로와 메뉴 경로를 `startsWith`로 비교해서 활성 메뉴를 표시해요. <br className="br" />
`/guide`는 `/inquiry`도 같은 섹션으로 처리해서 두 경로 모두에서 활성 상태가 유지돼요.

</FadeIn>

### 로그인 상태별 렌더링

| 상태                | 표시 영역                         |
| ------------------- | --------------------------------- |
| 비로그인            | 로그인 · 회원가입 · 견적요청 버튼 |
| 로그인 (일반 `U`)   | 알림 · 프로필 · 견적요청 버튼     |
| 로그인 (관리자 `A`) | 알림 · 프로필 버튼                |
| 로그인 (사업자 `B`) | 알림 · 프로필 버튼                |

### 탭 구성

| id  | 라벨       | 동작                           |
| --- | ---------- | ------------------------------ |
| 1   | 홈         | `/` 이동                       |
| 2   | 마이페이지 | ROLE에 따라 경로 분기          |
| 3   | 견적작성   | `/estimate` 이동 (강조 스타일) |
| 4   | 시공사례   | `/portfolio` 이동              |
| 5   | 전체메뉴   | MobileMenu 토글                |

## MobileMenu

<br />

<FadeIn>
NavBar의 전체메뉴 탭을 눌렀을 때 열리는 풀스크린 메뉴예요.

로그인 상태면 상단에 프로필 정보와 역할별 퀵메뉴가 표시돼요. <br className="br" />
비로그인이면 "로그인 해주세요" 버튼이 대신 나타나요. <br className="br" />

퀵메뉴는 `ROLE`에 따라 완전히 다른 항목을 보여줘요. <br className="br" />
일반회원은 나의 활동·견적 내역·혜택 현황, <br className="br" />
관리자는 통계·상담·일정·회원·배너 등 9개 메뉴, <br className="br" />
사업자는 포트폴리오·공사 현황·결제·혜택 4개 메뉴로 구성돼요.

</FadeIn>

<br />

### 역할별 퀵메뉴

| ROLE         | 항목                                                    |
| ------------ | ------------------------------------------------------- |
| `U` 일반회원 | 나의 활동, 견적 내역, 혜택 현황                         |
| `A` 관리자   | 나의 통계, 상담/일정/회원/배너/영상/결제/혜택/문의 관리 |
| `B` 사업자   | 포트폴리오, 공사 요청 및 현황, 결제 관리, 혜택 현황     |

---

## Props

### MobileMenu

| Prop      | Type         | Default | Description    |
| --------- | ------------ | ------- | -------------- |
| `isOpen`  | `boolean`    | -       | 메뉴 열림 여부 |
| `onClose` | `() => void` | -       | 닫기 핸들러    |

Header, Footer, NavBar는 props 없이 독립적으로 동작해요.
