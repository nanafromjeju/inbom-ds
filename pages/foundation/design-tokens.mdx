# Design Tokens

<br />

<FadeIn>
인봄 플랫폼 개발을 시작하기 전, <br className="br" /> 디자이너와 함께 색상·타이포그래피·간격을 먼저 정의했어요.

어떤 색이 어떤 상황에 쓰일지 미리 합의해두지 않으면,<br className="br" /> 개발하면서 **피그마와 코드 사이에 미묘한 불일치**가 생기거든요.<br className="br" />
비슷해 보이는 회색이 컴포넌트마다 조금씩 다른 값을 쓰거나,<br className="br" /> 에러 색상이 페이지마다 달라지는 식으로요.<br className="br" />

그래서 모든 값을 <span style={{ color: '#40E0D0' }}>CSS 변수로 중앙에서 관리</span>하는 구조를 잡았어요.<br className="br" /> 덕분에 디자이너가 색상을 수정하거나, 개발 중에 피그마와<br className="br" /> 불일치가 발견되더라도 토큰 하나만 바꾸면 전체에 반영됐어요.

</ FadeIn>

<FadeIn>

          height: '80px',

        }}
      />
      <div style={{ fontSize: '13px', fontWeight: 600 }}>{label}</div>
      <div style={{ fontSize: '12px', color: '#999' }}>{hex}</div>
    </div>
  ))}
</div>

## Z-index

레이어 우선순위를 토큰으로 관리해요.<br className="br" />
dropdown이 modal 위에 뜨는 실수를 방지하기 위해 명시적으로 값을 정의했어요.

| Token          | Value | 사용처              |
| -------------- | ----- | ------------------- |
| `--z-dropdown` | 100   | Select, 드롭다운    |
| `--z-modal`    | 200   | Modal, ConfirmModal |
| `--z-toast`    | 300   | Notification        |
| `--z-tooltip`  | 400   | Tooltip             |
